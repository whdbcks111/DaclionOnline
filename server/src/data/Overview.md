# Data Overview

`locations.json`의 25개 장소는 안전/중립/적대 위험도를 6/13/6곳으로 배치한다. 늘·화산·갱도 심부와 수정 왕좌는 적대 구역이다.

서버 시작 시 import 부작용으로 런타임 레지스트리에 들어가는 마스터 데이터를 둔다.

- `items.ts`: 아이템 이미지 key·`baseMetadata`·정의 태그·사용/공격 효과 등록. HP 회복은 `Entity.heal()`로 받는 치유량 상태효과를 반영하고 `restore_survival` 음식·음료는 배고픔/수분을 최대값 안에서 회복하며 `learn_skill` 스킬북은 신규 획득 성공 때만 한 개를 소비한다. `apply_status_effect`는 metadata의 효과 ID·레벨·시간을 사용하며 전투 강장제·비전 영약·신속의 물약이 60초 버프를 제공한다. 전용 아이콘 제작 전에는 기존 포션 fallback을 명시한다. 기본 미끼 조합에서 입질까지 약 31~45초가 걸리는 초보자 낚싯대·정교한 균형형 낚싯대, 보물상자 전용 범위/속도 특화 낚싯대, `/낚시` 시 묶음 자동 장착 지렁이 미끼와 `fishingCatalog.ts`의 물고기 아이템도 정의하며 인스턴스는 기본 metadata 전체를 복사하지 않고 top-level delta만 저장한다.
- `projectiles.ts`: 코드에서 참조하는 투사체 마스터 데이터. 물리 화살, 무탄약 지팡이 기본 공격용 마력 구체, 마법사 스킬 전용 마력탄을 각각 독립 정의한다.
- `monsters.ts`: `/몬스터정보`에 쓰는 설명과 Lv.1~50 몬스터의 속성 태그, 물리/마법 공격, 선택적 상태이상, 능력치·보상·드롭 정의. 일반 동급 경험치는 `level * 20`이며 슬라임 계열은 마지막 공격자형 저지능 AI, 수정맥의 군주는 치유 우선·도발 저항형 지능 AI 프로필을 가진다.
- `resources.ts`: 비공격 Entity 자원의 능력치, 공격 가능 여부, 도구 제한, 가중치 드롭, 범위 경험치, 상호작용·랜덤 쿨타임 정의. 일반/응축/수정 광맥과 1~2시간 쿨타임 및 각각 0.75% 가중치의 범위/속도 특화 낚싯대를 가진 보물상자가 있다.
- `shops.ts`: 음식·초급 장비·견습 마법 지팡이와 3종 전투 버프 물약을 파는 별등불 잡화점, 낚싯대·미끼 판매와 등급별 물고기 매입을 전담하는 물빛 연못 낚시상점, 은맥 광부 보급소에 연결되는 상점 태그·매매 목록·재고 정의.
- `fishingCatalog.ts`: 일반~신화 각 6종, 총 36종 물고기의 ID·이름·설명·중량·등급 단일 원본.
- `fishing.ts`: 공용 물고기 카탈로그를 낚시 보상 registry에 등록한다.
- `tagEffects.ts`: `source tag → target tag → modifier` 단방향 효과 테이블과 `/속성표`용 라벨·`affinities/*` 아이콘 key. 불·물·얼음·자연·독·전기·돌·어둠·빛·언데드·신성·벌레·금속·땅과 무생물을 모두 매핑한다.
- `statusEffects.ts`: 레거시 상태이상을 현재 source modifier·행동 제한·피해 API로 이식한 마스터 정의와 화염/빙결·둔화/신속 중화, 저항·해독·보존·발각 단방향 상호작용 표. 전용 아이콘 제작 전에는 의미가 가까운 기존 아이콘을 명시적으로 공유한다.
- `locations.json`: 루미나르 개척촌 광장·전직소·물빛 연못 낚시터, 바람결 초원, 안개수렁, 홍염산지, 보스 수정 왕좌를 포함한 미로형 피버릭 갱도의 좌표·연결·지도 랜드마크 `mapIcon`·지역별 대표 `mapColor`·통합 오브젝트·NPC ID·상점 배치 원본.
- `locations.ts`: Lv.10/20/28/36/45 연결 condition handler와 실제 이동 실패 시 공개할 필요 레벨 설명 등록.
- `progress.ts`: 게임 이벤트에 연결되는 공개 통계 정의. 현재 치명타 누적 횟수가 있다.
- `skills.ts`: 아이콘 key·스킬 마스터 metadata·계산 필드·포맷 문자열·Entity 공용 획득/발동/패시브 수명주기와 실제 계수 callback을 공유하는 밸런스 진단 metadata. 전투 본능·매의 눈·살의 감각·마력 순환 1차 패시브와 12개 조합별 엘리트 계승 패시브는 직업 조건이 유효할 때 source-key modifier를 적용하고 비활성 시 정리하며, 전용 아트 전까지 직업 아이콘을 fallback으로 쓴다. 계산된 현재 수치는 hover 계수 설명을 제공하며 실제 발동과 레벨 스케일 계산을 공유한다. 마력탄은 지팡이 기본 공격용 마력 구체가 아닌 전용 투사체 데이터를 사용한다. 직업 버프 6종은 현재 상승량·지속시간 성공 피드백을 제공한다. 치명타 5회에 획득하는 강타와 플레이어/광산 보스가 함께 쓰는 지각 붕괴가 있다.
- `jobs.ts`: 전사·궁수·암살자·마법사 1차 직업, 각 직업의 상시 패시브를 포함한 지급 스킬·메인/서브 modifier와 동일 직업을 제외한 순서 있는 12개 Lv.200 엘리트 조합 및 조합 전용 계승 패시브 지급.
- `crafting.ts`: predicate 재료·발견 조건·제작 시간·결과 factory 정의. 현재 철광석 3개와 돌 2개로 철 곡괭이를 만드는 `basic:iron_pickaxe`가 있다.
- `npcs.ts`: NPC ID, 조건부 진입점과 generator 대화 장면 정의. 안내인 리아와 슬롯별 전직 시험을 안내하는 전직관 세레나가 있다.
- `quests.ts`: NPC 제공/보고 ID, 단계형 목표와 lazy 표시명 보상 정의. `초원의 첫 의뢰`와 실제 태그 predicate를 명시하는 메인/서브별 8개 직업 전직 시험이 있다.

ID는 DB와 다른 데이터 파일에서 참조되므로 변경 시 참조 전체를 확인한다. 데이터 구조나 ID가 바뀌면 이 문서와 해당 [`docs/systems/`](../../../docs/systems) 문서를 갱신한다.
