# Commands Overview

PVP 조작은 `player.ts`의 `/대상지정p`(`tgp`)와 `/공격p`(`ap`)에서 순번·닉네임·`#고유번호`를 받고 Player의 지역/파티 공격 검증 API를 사용한다. `location.ts`의 `/위치`는 위험도·PVP 여부·대상 버튼을 함께 표시한다.

채팅에서 들어온 `/명령어` 또는 첫 단어가 등록된 별칭인 슬래시 없는 입력을 검증된 게임 동작으로 연결한다. 모든 명령은 `modules/bot.ts::registerCommand()`로 등록하고 `index.ts::initAllCommands()`가 분류별 초기화 함수를 호출한다.

- `general.ts`: 도움말, 권한별 등록 별칭을 보여주는 `/단축키목록`, 정보 열람 공개/비공개 모드, 랜덤, 공지, 관리자 eval. 도움말의 구분선은 공용 divider ChatNode, 인자·설명은 light/dark 공용 테마 token으로 출력한다.
- `player.ts`: 상태/인벤토리/아이템/장비/대상/공격/스탯. 아이템 사용은 `ActionType.ITEM_USE` 제한을 검사하고 실제 수량이 소비된 뒤 `item:used`를, 유효한 대상 지정은 `combat:target_selected`를 발행한다. 개수 선택 버리기 `/버리기 <슬롯> [개수]`·장착·대상·스탯 분배의 최소 단축은 `u/q/eq/t/r`이고 스탯 분배는 `st`도 유지한다. 상태창은 상세보기를 열기 전에도 EXP·위치·골드·직업 기본 정보를 표시하고 모든 영역 제목을 공용 divider로 구분한다. 상세 영역의 생명력은 타입별 보호막을 겹쳐 그리는 공용 health 노드를 사용한다. 자원과 능력치는 `AttributeType.icon`을 붙인 한 행 목록으로 출력해 긴 이름의 두 열 겹침을 방지한다. `/대상지정`과 `/공격` 자동완성은 생존 대상을 먼저, 사망·파괴 대상을 뒤에 표시한다. 상태창 맨 아래는 효과 아이콘·시간·hover 설명을, 장착/인벤토리는 아이템 아이콘과 내구도를 표시한다. 인벤토리 현재/최대 중량은 공용 formatter의 `kg` 단위를 사용한다.
- `karma.ts`: `/카르마(karma/krm)`로 자연 감소가 반영된 현재 값·클래스형 악명 단계·`🥀` 여부와 주요 제한 기준을 정보 공개 모드에 맞춰 표시한다.
- `patchNotes.ts`: `/패치노트(patchnotes/pn) [버전]`으로 최신 또는 지정 버전·날짜만 채팅에 요약하고, `[+]` 추가·`[/]` 수정·`[-]` 삭제 전체 기록은 스크롤 가능한 상세보기 안에 표시한다.
- `inspection.ts`: `/감정`의 인벤토리 번호·장착칸 해석과 `/몬스터정보`의 현재 장소 번호 조회. 감각 최소 요구치와 3단계 공개 범위를 검사하고 Item/Monster snapshot을 표시명·아이콘·게임 수치·해석된 효과로만 변환해 내부 ID·raw 태그·metadata를 숨긴다. 아이템 단위·합계 중량은 `kg`로 표시하고 같은 능력치의 고정/비율 modifier는 합산·곱산 요약해 한 행에 표시한다.
- `fishing.ts`: `/낚시(fish/f)`에서 낚시 서비스 공개 API를 호출하고, 미끼 미장착 시 인벤토리 묶음 자동 장착을 포함한 대기 시작 또는 장소·장비·미끼 오류를 본인에게 안내한다. `/낚시등급표(fishrate/fr)`는 현재 행운을 `getFishRarityChances()`에 적용한 등급별 실제 확률을 표시한다.
- `location.ts`: Monster/Resource 통합 위치 조회, 오브젝트 상호작용, 개별/전체 바닥 아이템 줍기(`p`)와 취소 가능한 이동. 바닥 아이템 목록·자동완성·획득 결과는 `Location.getDroppedItemDisplays()`와 Item 표시 API를 사용해 단조·명명 metadata가 적용된 인스턴스 이름을 노출한다. `/이동(mv/v)` 입력·시작·진행·도착 피드백은 항상 본인에게만 전송하며, `ActionType.LOCATION_TRAVEL` 제한을 검사하고 연결 순번·장소 ID·구분 기호 없는 이름·유일한 부분 이름을 허용한다. `/자동이동(nav/ago/av/amv)`은 표시 가능한 방문 장소를 검색해 현재 열린 연결의 A* 최단 거리 경로를 이동하고, `/이동취소(vc/mvc/goc)`는 단일 이동과 자동이동을 모두 중단한다. 잠긴 길을 실제 시도하면 공개 가능한 잠금 사유를 안내한다. `/위치(l/m)`는 정보 공개 모드를 따르고 제압된 오브젝트는 붉은 상태로 표시한다.
- `shop.ts`: 상점(`sh`) 조회, 구매(`bu`), 판매. 모든 경로가 Shop 카르마 접근 API와 Inventory predicate 수량·제거 API를 사용한다.
- `admin.ts`: 권한 10의 플레이어/월드 관리 명령. `/카르마설정`은 온라인·오프라인 Player의 공개 API와 dirty 저장을 사용하고, `/상태이상부여`는 온라인 대상만 조회해 `Entity.applyStatusEffect` 공개 API를 호출하며, `/레벨조정`은 `Player.adjustLevel`로 레벨 차이의 자동 스탯·분배 포인트까지 함께 증감한다.
- `balance.ts`: 권한 10의 `/스킬밸런스(sb)`, `/직업밸런스(jb)`, `/아이템밸런스(ib)`, `/밸런스프로파일(bp)`. 개별 계산 외에 레벨별 추천 장비와 동레벨 일반/보스, 평타+전체 스킬 공유 자원 로테이션을 실행해 DPS·처치 시간·사용량·실제 상대속도 회피율·회피 요구 민첩·관통 적용 뒤 유효 방어를 표시하며 `전체`는 공용 `BALANCE_PROFILE_LEVELS`의 Lv.20/50/75/100/140/180/200 구간을 비교한다.
- `progress.ts`: `/통계`로 공개 Progress snapshot을 공개 또는 비공개 표시한다.
- `ranking.ts`: `/순위 [카테고리]`(`rank`, `ranking`, `rk`)의 공동 순위·상위 5명 강조·고정 너비 열 UI와 `/순위공개`, `/순위비공개`, `/순위공개정보`의 전체 기본값·카테고리 예외 설정을 제공한다. 비공개 값은 순위·닉네임을 유지하고 수치만 `?`로 표시한다.
- `skill.ts`: `/스킬목록`의 보유 스킬 아이콘·패시브/발동/쿨다운 상태·정보 버튼 UI, `/스킬 이름` 발동과 `/스킬정보 이름`의 아이콘·레벨/경험치 진행도·계산 설명·소모값·개인/공유 재사용 대기시간·발동 조건 UI. 정보창은 `Skill.getInformationTagsSnapshot()`으로 공개 등록된 기술 계열과 속성을 나눠 표시하고 raw tag key와 미등록 구현 태그를 숨긴다. 패시브는 사용 버튼·사용 자동완성에서 제외하고 재사용 대기시간 없음으로 표시한다. 모든 명령은 SkillBook 공개 API만 사용하며 최소 단축어는 각각 `sl`, `k`, `si`다.
- `crafting.ts`: `/제작법목록(cl)`의 발견 제작법·재료·시간·버튼 UI와 `/제작 <이름> [개수](c)` 파싱/시작. 명령 metadata는 이름과 선택 개수를 두 인자로 표시하되, 실행 시에는 이름 중간 공백을 보존하고 마지막 숫자 token만 개수로 해석한다.
- `forging.ts`: `/단조 <형태> <제련 재료>(forge, fg)` 자동완성에 장검·도끼·단검·곡괭이·방패와 투구·흉갑·각반·철갑화를 제공하고, 메인/서브/계보의 대장장이 직업 또는 별도 금속 단조 스킬·소재 검증 뒤 리듬 단조 세션을 시작한다. 완료 메시지는 완성품 장비 레벨·품질·경험치를 표시한다. `/장비명명(gn)`은 감각 250 이상에서 자동 획득한 `장인의 명명`과 forge 제작자 ID를 검사해 직접 만든 장비만 이름을 바꾼다. `/마법부여(enc)`는 마도 대장장이의 보유 스킬·정신력·무기/중복 여부를 검사하고 성공 시 실제 화염·맹독·기절·빙결·실명 이름과 hover 상세를 표시한다. `/무기강화(rf)`는 전투 대장장이의 지핵 강화석을 소비해 +5까지 후가공한다.
- `puzzle.ts`: 상호작용으로 시작한 질문형 던전 장치에 `/퍼즐답 <정답>(pz)`을 제출하고 장소 이탈·만료 상태를 안내한다.
- `npc.ts`: `/대화 번호(tk)`, 선택지 버튼의 session 검증, `/대화종료`. 명령 입력은 숨기며 모든 상태 변경은 NpcDialogue 공개 API를 사용한다.
- `quest.ts`: `/퀘스트목록(ql)`, `/퀘스트정보(qi)`, `/퀘스트포기(qa)`의 단계 목표·보상·상태 UI와 자동완성. QuestBook snapshot·명령형 API만 사용한다.
- `affinity.ts`: `/속성표(affinity)`에서 TagEffect 표시 snapshot의 내부 tag key를 제외하고 표시명·아이콘·배율만 전체 화면 스크롤 UI로 렌더링한다. 속성 아래 공격/방어를 1단계로, 우세·열세·무효/취약·저항·면역을 각각 별도 한 줄의 2단계로 렌더링한다.
- `map.ts`: `/지도(map)`에서 플레이어별 방문 장소와 한 단계 인접 미방문 장소를, 관리자 `/전체지도(fullmap)`에서 hidden을 포함한 전체 월드를 `worldMap` 상세보기 메시지 노드로 전달한다.
- `career.ts`: `/직업(job, career)` 현재 메인·서브·엘리트 상세 UI와 `/직업정보(ji)` 직업 마스터 정보·지급 스킬 UI.
- `party.ts`: `/파티초대/수락/거절/정보/나가기/해산/강퇴` 진입점. 기본 조작 단축은 `pi/pa/pd/pt/pl/pb`다. 고유번호·닉네임 자동완성과 PartyManager 공개 API만 사용하며 구성 변경 알림은 당사자에게 비공개로 보낸다.
- `trade.ts`: `/거래요청(tr)`과 수락/거절, 아이템 추가·회수, 제안 골드 총액 설정, 양쪽 확인·취소 명령. TradeManager의 불변 snapshot/event만 사용하며 제안이 바뀔 때 양쪽의 이전 비공개 카드를 삭제하고 최신 확인 상태·버튼이 있는 카드 한 장으로 교체한다.
- `tutorial.ts`: `/튜토리얼시작`, `/튜토리얼스킵`, 행동 없는 확인 단계용 `/튜토리얼다음`, 낚시·광질·사냥 실제 체험 순서를 정하는 `/튜토리얼선택`. 콘텐츠 완료는 명령이 아니라 물고기 획득·광석 파괴·몬스터 처치 이벤트로 판정하며 모든 상태와 보상은 Tutorial/QuestBook/Inventory/SkillBook 공개 API를 호출한다.

도움말·단축키목록·패치노트·상태창·인벤토리·감정·몬스터정보·통계·순위·순위공개정보·스킬/퀘스트/직업 정보·목록·상점·제작법·위치·지도·속성표·낚시등급표·파티정보는 `information: true`로 등록한다. 공개 모드에서는 입력과 결과가 현재 채널에 보이고, 조작 명령은 이 설정의 영향을 받지 않는다.

명령 이름·인자·권한·자동완성·표시 방식이 바뀌면 이 문서와 [`docs/systems/chat-command.md`](../../../docs/systems/chat-command.md)를 갱신한다.
