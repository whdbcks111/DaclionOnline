# 레거시 프로젝트 레퍼런스

## 지정된 레거시 버전

- 저장소: [whdbcks111/LucadionTextRpg-React](https://github.com/whdbcks111/LucadionTextRpg-React)
- 관계: DaclionOnline보다 앞서 제작된 텍스트 RPG 채팅 프로젝트로 보고, 이전 세대의 기획과 사용자 경험을 확인하는 레거시 레퍼런스로 사용한다.

## 참고 범위

다음 내용을 설계할 때 과거에 시도한 방향과 플레이 흐름을 파악하는 자료로 참고할 수 있다.

- 텍스트 RPG의 채팅 중심 상호작용과 화면 흐름
- 지역, 몬스터, 아이템, 성장 등 콘텐츠 구성 아이디어
- 명령어 이름, 안내 문구, 플레이 편의 기능의 과거 형태
- 현재 프로젝트에서 기능을 새로 설계할 때 유지하거나 개선할 경험적 요소

## 적용 원칙

레거시 저장소는 참고 자료이며 현재 구현의 원본(source of truth)은 아니다.

1. 현재 DaclionOnline의 `docs/`, 폴더별 `Overview.md`, 공개 API, 마스터 데이터와 DB 스키마를 항상 우선한다.
2. 레거시 코드를 그대로 복사하거나 레거시 구조에 맞추지 않는다. 필요한 개념만 현재의 서버 권위형 클라이언트·서버 구조와 개발 원칙에 맞게 다시 설계한다.
3. 같은 기능이 현재 프로젝트에 이미 있으면 현재 공개 API를 재사용하고, 레거시 구현은 동작 의도와 UX를 비교하는 용도로만 본다.
4. 레거시와 현재 동작이 다르면 명시적인 작업 요구사항과 현재 문서를 기준으로 판단한다.
5. 레거시에서 아이디어를 가져와 의미 있는 시스템 동작을 추가했다면 관련 현재 문서에 채택한 개념과 현재 방식의 차이를 남긴다.

## 현재 채택한 PVP 개념

레거시의 `대상지정p`/`공격p`와 평화·중립·일반 구역 구분을 채택했다. 현재 프로젝트에서는 이를 `safe | neutral | hostile` 위험도와 `Player.getAttackDeniedReason()`으로 재구성해 직접 공격·스킬·투사체가 하나의 서버 판정을 공유한다. 레거시의 큰 카르마 수치에 따른 랜덤 장비 드롭은 즉시 복제하지 않고, 예측 가능한 지역별 경험치·골드·부활 차이와 PVP 처치 통계를 먼저 도입했다. 세부 규칙은 [PVP·지역 위험도](systems/pvp-regions.md)를 따른다.
