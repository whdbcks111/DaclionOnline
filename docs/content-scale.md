# 콘텐츠 규모 추적

레거시 저장소의 콘텐츠 밀도를 현재 서버 권위형 구조로 옮길 때 사용하는 수량 기준과 진행 상태다. 레거시의 이름·코드를 그대로 복사하지 않고, 현재 태그·상태효과·위협 AI·퍼즐·상점·퀘스트 API로 변형한다.

## 2026-07-22 기준

| 분류 | 레거시 정적 정의 수 | 현재 런타임 레지스트리 | 판단 |
| --- | ---: | ---: | --- |
| 장소 | 233 | 113 | 가장 큰 격차. 권역별 분기·합류와 플레이 동선을 포함해 계속 확장한다. |
| 몬스터 | 65 | 83 | 양적 기준 도달. 이후에는 신규 장소의 레벨·속성·보스 기믹 다양성을 우선한다. |
| 아이템 | 169 | 149 | 지역 소재·소모품·직업 장비·전승서 묶음으로 격차를 줄이는 중이다. |
| 상태효과 | 35 | 46 | 양적 기준 도달. 신규 효과 수보다 속성 상쇄·행동 제한·설명 정확성을 우선한다. |
| 상점 | 9 | 8 | 레거시 수량까지 지역 경제 한 곳이 더 필요하다. |
| 제작법 | - | 27 | 고정 조합법과 대장장이 단조를 병행한다. |
| 스킬 | - | 119 | 성장·전승·엘리트 스킬과 보스 공용 SkillData를 함께 센다. |
| 자원 | - | 28 | 광맥·보물·퍼즐·보스 기믹 오브젝트를 포함한다. |
| 퀘스트 | - | 20 | 직업 전직과 지역 연속 퀘스트를 포함한다. |
| NPC | - | 7 | 장소는 정의 전체가 아니라 NPC ID만 참조한다. |

현재 추가가 완료된 선택 권역은 은빛그물 숲, 황혼왕릉, 유리모래 사막, 서리잔향 설원·빙경궁, 안개파도 해안·침몰왕도다. 각 권역은 단순 장소 수가 아니라 사냥 소재, 지역 상점, 제작법, 탐색 퍼즐, 가중치 보물, NPC 연속 퀘스트와 최소 하나의 차별화된 보스를 함께 갖춘다. 안개파도 권역은 Lv.156~186의 두 갈래 해안과 두 갈래 수몰 왕도, 세이렌 군주와 침몰제독을 연결한다.

## 확장 우선순위

1. 장소 격차를 줄이되 일자형 복도를 피하고 안전 거점·분기 사냥터·숨은 길·다음 권역 합류를 한 단위로 구현한다.
2. 신규 일반 몬스터는 기존 72종을 무의미하게 복제하지 않고, 부족한 레벨대·속성·상태이상·AI 성향 조합을 채운다.
3. 권역마다 소재·소모품·직업별 주무기/보조장비를 추가해 아이템 169개 이상을 먼저 달성하고, 상점은 최소 레거시 수량 9곳을 넘긴다.
4. `balance:report`의 동레벨 일반/보스 프로필에서 회피율·관통 후 방어·처치시간·평타 비중을 확인한다. 신규 장비는 같은 구간의 이전 장비보다 의미 있게 강하되 직업 로테이션을 대체하지 않게 조정한다.
5. 코드 콘텐츠가 목표 규모와 플레이 동선을 충족한 뒤에만 임시 fallback 목록을 확정하고 전용 아이콘을 병렬 생성한다.

## 검증 기준

- `server/src/data/world.test.ts`가 장소 양방향 연결, 오브젝트 ID, 권역 대표색, 상점·제작법·퀘스트·퍼즐·보물·보스 패턴을 권역 단위로 검증한다.
- `server/src/scripts/validateMasterData.ts`가 모든 참조와 명시된 fallback 아이콘 경로를 검증한다.
- `server/src/scripts/reportBalance.ts`가 황혼왕릉·유리모래·빙경궁·침몰왕도 장비를 같은 직업/레벨 기준선에서 비교한다.
- 수량은 문서의 추정이 아니라 현재 마스터 데이터를 실제로 import한 런타임 레지스트리 수와 `locations.json` 길이로 갱신한다.
