# 콘텐츠 규모 추적

레거시 저장소의 콘텐츠 밀도를 현재 서버 권위형 구조로 옮길 때 사용하는 수량 기준과 진행 상태다. 레거시의 이름·코드를 그대로 복사하지 않고, 현재 태그·상태효과·위협 AI·퍼즐·상점·퀘스트 API로 변형한다.

## 2026-07-23 기준

| 분류 | 레거시 정적 정의 수 | 현재 런타임 레지스트리 | 판단 |
| --- | ---: | ---: | --- |
| 장소 | 233 | 209 | 남은 한 개 후반 권역에서 분기·합류와 플레이 동선을 포함해 목표 규모에 도달한다. |
| 몬스터 | 65 | 127 | 양적 기준 도달. 이후에는 신규 장소의 레벨·속성·보스 기믹 다양성을 우선한다. |
| 아이템 | 169 | 220 | 양적 기준 도달. 이후에는 지역 획득·소비 경로와 역할별 성능 차이를 우선한다. |
| 상태효과 | 35 | 46 | 양적 기준 도달. 신규 효과 수보다 속성 상쇄·행동 제한·설명 정확성을 우선한다. |
| 상점 | 9 | 12 | 양적 기준 도달. 권역별 소재 회수와 장비 공급 동선을 유지한다. |

| 퀘스트 | - | 30 | 튜토리얼, 직업 전직과 지역 연속 퀘스트를 포함한다. |
| NPC | - | 11 | 장소는 정의 전체가 아니라 NPC ID만 참조한다. |

현재 추가가 완료된 선택 권역은 은빛그물 숲, 황혼왕릉, 유리모래 사막, 서리잔향 설원·빙경궁, 안개파도 해안·침몰왕도, 역설기계고, 잿빛성흔 심연, 공허왕관 성채, 월식해구·백야성소다. 각 권역은 단순 장소 수가 아니라 사냥 소재, 지역 상점, 제작법, 탐색 퍼즐, 가중치 보물, NPC 연속 퀘스트와 최소 하나의 차별화된 보스를 함께 갖춘다. 월식해구는 Lv.310~345의 24개 분기 장소, 순차 해일·수압 패턴의 월조 리바이어던과 무작위 백야·월식 패턴의 대사제, 조류제단 뒤 숨은 보물고와 파괴 전 대사제 피해를 65% 줄이는 조류거울을 연결한다.

## 확장 우선순위

1. 장소 격차를 줄이되 일자형 복도를 피하고 안전 거점·분기 사냥터·숨은 길·다음 권역 합류를 한 단위로 구현한다.
2. 신규 일반 몬스터는 현재 정의를 무의미하게 복제하지 않고, 부족한 레벨대·속성·상태이상·AI 성향 조합을 채운다.
3. 아이템·상점 수량 기준은 달성했으므로 신규 권역에서는 기존 소재와 겹치지 않는 획득처·소비처, 전승 기술과 장비 역할을 우선한다.
4. `balance:report`의 동레벨 일반/보스 프로필에서 회피율·관통 후 방어·처치시간·평타 비중을 확인한다. 신규 장비는 같은 구간의 이전 장비보다 의미 있게 강하되 직업 로테이션을 대체하지 않게 조정한다.
5. 코드 콘텐츠가 목표 규모와 플레이 동선을 충족한 뒤에만 임시 fallback 목록을 확정하고 전용 아이콘을 병렬 생성한다.

## 검증 기준

- `server/src/data/world.test.ts`가 장소 양방향 연결, 오브젝트 ID, 권역 대표색, 상점·제작법·퀘스트·퍼즐·보물·보스 패턴을 권역 단위로 검증한다.
- `server/src/scripts/validateMasterData.ts`가 모든 참조와 명시된 fallback 아이콘 경로를 검증한다.
- `server/src/scripts/reportBalance.ts`가 황혼왕릉·유리모래·빙경궁·침몰왕도·역설기계고·잿빛성흔 심연·공허왕관·월식해구 장비를 같은 직업/레벨 기준선에서 비교하고, 지역 전승 기술의 회피·관통·자원·쿨다운을 함께 출력한다.
- 수량은 문서의 추정이 아니라 현재 마스터 데이터를 실제로 import한 런타임 레지스트리 수와 `locations.json` 길이로 갱신한다.
