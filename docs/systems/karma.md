# 카르마·악명 시스템

레거시의 중립 지역 PVP 카르마와 고카르마 부활 지연을 현재 서버 권위형 PVP·지역 위험도 위에 재구성한다. 카르마는 Player가 소유하며 다른 기능은 `Player.addKarma/reduceKarma/setKarma`, `getKarmaSnapshot`, `getKarmaAccessDeniedReason` 공개 API만 사용한다.

## 저장과 자연 감소

`players.karma`는 저장 기준값, `players.karma_updated_at`은 그 기준 시각이다. `KarmaState`는 둘 사이의 경과 시간을 읽을 때 계산해 초당 `0.003`, 시간당 `10.8`만큼 감소시킨다. 따라서 매 tick 값을 변경하거나 DB에 쓰지 않아도 온라인·오프라인 시간이 같은 규칙으로 반영된다.

명시적인 악행·감소·관리자 설정은 먼저 경과 감소를 현재 값에 반영하고 새 기준값과 시각을 만든 뒤 Player를 dirty 처리한다. 기존 30초·unload·종료 저장에서 함께 flush하며 변경 시 `player:karma_changed` 이벤트를 발행한다.

## 단계와 정책 API

`KarmaTier`는 표시명과 채팅 표식 기준을 가진 클래스형 enum이다.

| 최소 카르마 | 단계 | 의미 |
| ---: | --- | --- |
| 0 | 무고 | 별도 제한 없음 |
| 25 | 흔적 | 악행의 흔적이 남은 상태 |
| 100 | 악명 | 현상 대상, 채팅 `🥀`, 강화 사망 패널티 기준 |
| 300 | 무법자 | 일부 질서 시설 거래 제한 기준 |
| 1000 | 재앙 | 최고 악명 표시 단계 |

`KarmaAccessPolicy`는 선의 성향 의뢰 `100`, 질서 시설 `300`, 교단 성소 `500`의 독립 제한과 사용자용 거부 문구를 제공한다. 각 NPC·상점·퀘스트는 raw 수치를 직접 비교하지 않고 이 정책과 태그를 사용한다.

## PVP·사망 계산 경계

`getPvpKarmaGain()`은 현상 대상이 아닌 플레이어 처치에 중립 구역 `25`, 적대 구역 `10`을 반환한다. 카르마 `100` 이상인 현상 대상을 처치하면 카르마 대신 `getKarmaHeroReward()`가 영웅 효과 레벨과 15~60분 지속시간을 계산한다.

`getKarmaDeathPenalty()`은 카르마 `100`부터 지역 기본 손실에 더할 경험치·골드 손실률, 고정 카르마 감소 `15`, 레거시와 같은 `5분 + karma^1.5/10`의 추가 부활 대기시간(최대 12시간)을 반환한다. 실제 적용과 사용자 알림은 Player 사망 경계에서 한 번만 실행하며 재접속 복원은 다시 적용하지 않는다.

`modules/karma.ts::initKarma()`가 `combat:pvp_kill`을 구독한다. 일반 피해자 처치는 `Player.addKarma()`를 호출하고, 현상 대상 처치는 `hero` 상태효과를 적용한다. 영웅 효과는 레벨에 따라 획득 경험치를 15~35% 높이며 상태효과 공통 규칙대로 런타임에서 갱신·제거된다.

## 사용자 확인

- `/카르마` (`karma`, `krm`): 현재 값·단계·자연 감소량·주요 제한 기준을 표시한다.
- `/상태창`: 기본 정보에 현재 카르마와 단계, 악명 상태의 `🥀`를 표시한다.

운영 DB에는 `20260723000000_add_player_karma` migration이 필요하다.
