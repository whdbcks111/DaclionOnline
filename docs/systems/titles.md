# 칭호 시스템

칭호는 특정 행동·스탯·직업 조건을 달성해 영구 획득하고 한 번에 하나만 장착하는 업적형 성장 요소다. 장착한 칭호는 채팅 닉네임과 상태 HUD·상태창에 표시되며, 조건에 맞는 고유 패시브 하나를 제공한다.

## 구조

```text
GameEvent
  └─ defineStatistic(title-stat:*)
       └─ PlayerProgress counter
            └─ TitleBook.refreshAcquisitions()
                 ├─ title-owned:* flag
                 ├─ title-blocked:* admin revoke flag
                 └─ title:equipped state

TitleBook
  ├─ 조건부 Attribute modifier / 경험치 배율
  └─ CombatPipeline 준비·피해 후 hook
```

- `models/Title.ts`는 `TitleData` 레지스트리와 플레이어별 `TitleBook`을 소유한다.
- `data/titles.ts`는 획득 통계, 레거시 이식 칭호와 신규 칭호, 전투 hook을 등록한다.
- 소유 여부와 장착 ID는 기존 `PlayerProgress`에 저장하므로 별도 Prisma 모델이나 마이그레이션이 필요 없다.
- 기능 코드는 Progress key나 칭호 레지스트리를 직접 수정하지 않고 `TitleBook.grant/revoke/equip/unequip`을 사용한다.

## 획득과 장착

`TitleBook`은 Progress 변경 직후와 최대 1초 간격으로 미획득 칭호의 `canAcquire(player)`를 평가한다. 신규 획득 시 채팅과 notification으로 `칭호 [ 이름 ] 를 획득했습니다!`를 알리고 `title:acquired` 이벤트를 발행한다.

- `/칭호목록`(`tl`): 획득한 칭호, 획득 조건, 패시브, 현재 장착/활성 상태를 표시한다.
- `/칭호장착 <이름 또는 번호>`(`te`, `tu`): 보유 칭호를 장착한다.
- `/칭호장착해제`(`tue`): 현재 칭호를 해제한다.

목록은 정보성 명령이므로 플레이어의 정보 공개/비공개 모드를 따른다. 장착 명령은 조작 명령이며 입력을 채팅에 노출하지 않는다.

권한 10 관리자는 `/칭호부여 <대상> <칭호>`와 `/칭호삭제 <대상> <칭호>`를 사용할 수 있다. 관리자 페이지의 `스킬·효과` 카테고리에서도 미보유 칭호 부여와 보유 칭호 삭제를 검색 가능한 선택창으로 제공한다. 삭제는 장착 패시브까지 즉시 해제하고 `title-blocked:*`를 남겨 이미 달성한 조건으로 곧바로 자동 재획득되는 것을 막는다. 같은 칭호를 관리자가 다시 부여하면 차단 flag가 제거된다.

## 패시브 적용

지속 패시브는 `title:equipped` source 하나로 적용한다. 칭호를 바꾸거나 대상·장비 조건이 달라지면 이전 source의 능력치와 경험치 배율을 먼저 제거한 뒤 새 효과만 적용한다.

- 대상 조건형 칭호는 살아 있고 같은 장소에 있는 현재 지정 대상의 태그를 검사한다.
- 늑대 학살자는 `entity:wolf` 대상 지정 중 공격력·마법력 +5%를 제공한다.
- 곡괭이 살해자와 액스 파이터처럼 적중 효과가 있는 칭호는 `CombatPipeline`의 실제 피해 성공 뒤에만 기절·출혈을 시도한다.
- 직접 공격과 소유 투사체는 최종 `attackOwner`의 장착 칭호를 사용한다. 곡괭이·도끼 직접 적중 효과는 투사체에는 적용하지 않는다.
- 현재 칭호 표시명은 메시지 생성 시점에 `ChatMessage.equippedTitle`로 스냅샷되므로 이후 장착을 바꿔도 기존 메시지 표시는 변하지 않는다.

## 콘텐츠

레거시에서 실제 획득 가능했던 칭호 16종을 현재 스탯 규모·태그·상태효과 API에 맞춰 이식했다.

- 사냥: 늑대 학살자, 언데드 킬러/슬레이어, 벌레 사냥꾼, 불꽃 수집가
- 전투 숙련: 곡괭이 살해자, 액스 파이터, 격투가, 광부의 길
- PVP: 학살자(유효 플레이어 처치 100회), 몰살자(유효 플레이어 처치 500회)
- 성장·직업: 아크스펠, 마도사, 속사, 페이탈디드, 초감각

신규 칭호는 슬라임 연구가, 거인에게 맞서는 자, 광맥을 읽는 자, 전설을 낚은 자, 불꽃과 망치의 주인, 삼원소 조율자다. 레거시의 `과거의 영광`은 원본에서도 획득 조건이 항상 거짓이어서 이식 대상에서 제외했다.

PVP 유효 처치는 양쪽 누적 플레이 1시간 이상, 레벨 차이 30 이하, 피해자 부활 후 60초 경과, 같은 공격자→피해자 24시간 1회 조건을 모두 만족해야 한다. 실제 처치·사망·카르마는 그대로 처리하고 `combat:pvp_credited_kills` 칭호 진행도만 제외해 부계정 반복 처치로 칭호를 올릴 수 없게 한다.

## 확장 규칙

새 칭호는 다음 순서로 추가한다.

1. 기존 공개 통계로 조건을 표현할 수 있는지 먼저 확인한다.
2. 새 누적값이 필요하면 `GameEvent` 성공 지점과 `defineStatistic()`을 연결한다.
3. `defineTitle()`에 사용자용 획득 조건과 정확한 수치가 들어간 패시브 설명을 쓴다.
4. 대상 분류는 이름이나 Monster ID가 아니라 `GameTags`를 사용한다.
5. 능력치 효과는 source modifier, 공격 적중 효과는 CombatPipeline callback으로 구현한다.
6. 획득·영속 복원·조건부 활성/해제 테스트와 패치노트를 같은 변경에 포함한다.
