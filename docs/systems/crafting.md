# 제작법·제작 시스템

제작법은 `models/Crafting.ts`의 정적 레지스트리에 등록하며, 제작법 발견 상태는 기존 `PlayerProgress` flag로 저장한다. 재료 판정과 교환은 제작 코드가 Inventory raw 배열을 읽지 않고 `Inventory.selectItems/replaceSelectedItems`를 사용한다.

```text
data/crafting.ts::defineCraftingRecipe
  ↓
Player.update → updateCraftingRecipeDiscovery → PlayerProgress flag
  ↓
/제작법목록 또는 /제작 <이름> [개수]
  ↓
Inventory.selectItems → 제작 대기 → factory → replaceSelectedItems
```

## 제작법 정의

`CraftingRecipeData` 필드는 다음과 같다.

| 필드 | 의미 |
| --- | --- |
| `id` | `namespace:path` 형식의 영구 ID |
| `resultItemDataId` | 일반적인 결과 아이템 ID. `name`을 생략하면 이 아이템의 이름을 제작법 이름으로 쓴다. |
| `name`, `aliases`, `description` | 명령 검색과 UI 표시 정보 |
| `ingredients` | `CraftingRecipeIngredient` 배열. label, 필요 수량, `Item` predicate를 가진다. |
| `craftTime` | 1개 제작에 걸리는 초. 여러 개는 `craftTime × 개수`만큼 걸린다. |
| `create` | 플레이어, 제작법, 개수, 실제 선택된 재료를 받아 `ItemSnapshot` 결과를 만드는 factory |
| `discoveryCondition` | 선택. 생략하면 1회분 재료를 모두 소지했을 때 발견한다. 정의하면 기본 조건을 완전히 대체한다. |
| `tags` | 제작법 분류와 후속 조건에 쓸 태그 |

정의 ID로만 필터링할 때는 `CraftingRecipeIngredient.item(itemDataId, count)`를 쓴다. 아이템 상태를 보는 재료는 직접 predicate를 정의한다.

```ts
new CraftingRecipeIngredient(
    '내구도 50% 이상의 검',
    1,
    item => item.category === '검'
        && item.durability !== null
        && item.baseDurability !== null
        && item.durability >= item.baseDurability * 0.5,
)
```

factory의 `ingredients` 안에는 predicate에 실제로 선택된 `Item`과 수량이 들어온다. 따라서 수리, 합성, 메타데이터 승계처럼 재료 인스턴스의 상태에 따라 결과가 달라지는 제작을 만들 수 있다.

## 재료 선택과 소비

`Inventory.selectItems()`는 최대 유량으로 여러 predicate와 아이템 수량을 매칭한다. 하나의 아이템이 여러 재료 조건에 맞더라도 같은 수량을 중복 계산하지 않으며, 가능한 전체 배정이 있으면 찾는다.

`Inventory.replaceSelectedItems()`는 선택 수량, 결과 snapshot, 제작 후 총 무게를 먼저 검증한 뒤 재료 제거와 결과 추가를 하나의 도메인 작업으로 처리한다. 제작 대기 중에 재료가 달라지면 완료 시점에 다시 선택하고, 조건을 충족하지 못하면 소비 없이 실패한다.

## 발견·실행 수명주기

- 발견 상태는 `crafting:recipe/{namespace}/{path}` Progress flag로 저장되며 플레이어의 주기 저장, unload, 종료 flush에 포함된다.
- `Player.update()`가 0.5초 마다 미발견 제작법 조건을 검사하며, 발견 시 채팅과 notification을 보낸다.
- 한 플레이어는 동시에 하나의 제작만 진행한다. 접속 종료나 사망 시 취소된다.
- 재료는 시작 시 잠그거나 소비하지 않고 완료 시 교환한다. 따라서 중도 취소로 재료가 유실되지 않는다.
- 제작법 발견과 제작 완료는 각각 `crafting:recipe_discovered`, `crafting:item_crafted` GameEvent를 발행한다.

## 명령어와 현재 콘텐츠

- `/제작법목록`: 발견한 제작법의 재료, 1개당 시간, 현재 제작 가능 여부와 제작 버튼을 표시한다.
- `/제작 <제작법이름> [개수]`: 마지막 token이 숫자면 1~99의 제작 개수로 해석하고 생략 시 1개를 만든다.
- `basic:iron_pickaxe`는 철광석 3개와 돌 2개로 4초에 철 곡괭이 1개를 만든다.
- 황혼왕릉의 5개 자동 발견 제작법은 지역 소재와 기존 제련 소재를 함께 사용해 묘지기 향약, 맹세철 장검, 진혼 시위, 애도목 지팡이, 묘문 수호방패를 만든다. 상점 구매 외에도 사냥·제련·제작을 잇는 회수 경로다.
- 유리모래 사막의 6개 자동 발견 제작법은 지역 소재로 그늘 물통과 모래쐐기 장검·태양실 활·신기루 독아·태양유리 지팡이·태양거울 방패를 만든다. 재료 predicate는 공개 아이템 API와 데이터 ID를 사용하고 결과는 기존 마스터 아이템 factory로 생성한다.
- 서리잔향 설원·빙경궁의 7개 자동 발견 제작법은 눈솔이끼·상고 수정·경철·빙실·극광·빙결 핵으로 빙결 저항/재생 소모품과 빙맥 절단검·빙실 연궁·경빙 송곳니·극광분광 지팡이·빙경 성벽방패를 만든다. 상점과 보스 드롭만이 아니라 사냥·채굴·제작을 통한 회수 경로를 함께 제공한다.
- 안개파도 해안·침몰왕도의 8개 자동 발견 제작법은 해무 소금·흑산호·해무비늘·조류진주·침수 군단 휘장·심해철·청해초 수지·해수룡 골편으로 생존 소모품 3종과 파식 조류검·해무 조류궁·흑산호 침·심해진주 지팡이·침몰제독 방패를 만든다. 해안 채집, 두 보스 드롭, 항구 상점과 제작이 같은 지역 소재를 순환한다.
- 역설기계고의 9개 자동 발견 제작법은 시간강 파편·기억 톱니·광자 렌즈·공허 용수철·논리핵·역설 실·자동인형 장갑판·균열 수정을 사용한다. 태엽 작업식·위상 촉진제·논리회로 영약·시간봉합 연고와 역설절단검·광자연사궁·공허태엽 단검·논리핵 지팡이·인과율 방패를 만들며, 중계소 상점과 사냥·채굴·유물함 보상이 같은 소재 순환을 이룬다.
- 잿빛성흔 심연의 8개 자동 발견 제작법은 잿빛 힘줄·흑염 잔재·공허뿔·저주뼈·밤쇠·재왕 인장·심연가죽·애도의 눈을 사용한다. 생존 소모품 3종과 재가름 장검·공허뿔 장궁·황혼송곳·흑염각 지팡이·재성벽 방패를 만들며, 휴게소 상점·밤쇠 채굴·보스 사냥·봉인 유산고가 같은 소재 순환을 이룬다.

## 조합형 단조 장비 기반

`Forging`은 일반 제작법과 달리 하나의 고정 마스터 아이템을 그대로 복제하지 않는다. 장검·도끼·단검·방패·곡괭이 `ForgeForm`과 철·금·루비·에메랄드·다이아몬드·홍염강 `ForgeMaterial`을 조합하고, 리듬 단조 품질로 효율을 계산한다. 주 능력치는 형태의 기준 수치에 제작자 레벨 10 초과분, 감각 50 초과분, 최대 200%의 제련 정밀도를 `craftsmanshipPower`로 환산한 뒤 형태·재료·정확도·성향을 곱한다. 따라서 Lv.200·감각 1000·제련 정밀도 150%의 균형 잡힌 철 장검은 정확도 85%에서 약 480 공격력을 가지며, 같은 조건의 명품 다이아몬드 장검은 600 이상의 공격력으로 성장한다. 홍염강은 홍염산지의 화맥 광맥에서 캔 `화맥 광석`을 마력 제련해야 얻는 지역 전용 합금이며 불·금속 태그, 마법력과 치명타 피해 보너스를 가진다. 형태와 재료는 기본 공격/방어 수치·내구도·속성 태그를 결정한다. 서버 난수로 선택하는 단조 성향은 단순 수식어가 아니라 능력치 trade-off다. 예리함은 위력/내구도 대신 치명타, 중량형은 공격/이동속도 대신 위력/내구도, 정밀형은 위력 대신 관통/공격속도, 질긴 성향은 위력 대신 방어/내구도, 마도형은 물리 위력 대신 마법 능력, 불안정형은 내구도 대신 최고 위력을 얻는다. 방패·곡괭이는 무기 전용 치명타 성향에서 제외된다. 이 성향과 별도로 16% 확률의 특이 각인이 발생한다. 과부하(속도↔위력), 파갑(위력↔관통), 필살(치명타율↔치명타 피해), 생명 결속(위력↔레벨 비례 생명력/방어), 마도 결속(위력↔감각 비례 마법력/관통) 중 하나이며 이름 앞 단어와 `forge.quirk` metadata에도 기록된다. 일반 제작품은 높은 확률로 `철 장검` 같은 재료+형태 이름을 사용하고, 성향이 있으면 `불안정한 철 장검`처럼 평범한 이름에도 표시된다. 나머지는 품질/성향이 고른 판타지 접두어 어휘군, 실제 재료명, 형태별 RPG 명칭 후보를 조합한다. 완벽한 품질은 형태 고유 접두어를 사용하므로 품질 100%의 다이아몬드 도끼는 `익스클리프 다이아몬드 액스`가 된다.

결과는 `forged_*` 형태별 마스터 템플릿을 사용하되 `customName`, `customDescription`, `maxDurability`, `instanceModifiers`, `forge` metadata delta와 재료 속성 영속 태그만 저장한다. 따라서 같은 아이템 데이터 ID여도 서로 다른 이름·내구도·능력치를 갖는다. 공격/방어 수치는 형태 기본값 × 재료 배율 × 리듬 효율 × 제작 숙련 배율로 계산한다. 제작 숙련은 레거시 단조의 감각 100 초과분 효율을 현재 수치 체계에 맞춰 완화한 값으로, 캐릭터 레벨·감각·제련 정밀도를 반영하며 레벨/감각/정밀도별 상한을 둔다. 내구도에는 숙련 배율 증가분의 40%만 반영한다. `Item` 공개 getter와 `Equipment`가 이 유효값을 사용하며 metadata 원본이나 장비 내부 row를 직접 읽지 않는다. 전용 아이콘 제작 전에는 각 형태의 기존 카테고리 아이콘을 fallback으로 사용한다.

대장장이 직업 또는 호환 계보를 가진 플레이어가 감각 250을 달성하면 `장인의 명명`을 자동 획득한다. `/장비명명 <인벤토리 번호|장착칸> <새 이름>`은 대상의 `forge.creatorUserId`를 공개 도메인 API에서 검사하므로 본인이 직접 단조한 장비만 공백 포함 2~24자의 이름으로 바꿀 수 있다. 이름은 `customName` delta에 저장되어 해당 인스턴스에만 적용되고 Inventory/Equipment의 기존 dirty flush로 영속된다.

대장장이 메인 + 마법사 서브가 Lv.200에 `마도 대장장이`가 되면 `마법 부여`를 얻는다. `/마법부여 <인벤토리 번호|장착칸>`은 정신력 80을 쓰고 아직 마법이 없는 무기에 한 번만 화염·맹독·기절·빙결·실명 계열 적중 효과를 부여한다. 스킬 설명은 다섯 후보를 이름으로 열거하고 hover에 지속 피해·치유 감소·행동 제한·속도 감소를 설명한다. 성공 메시지도 `효과` 같은 익명 표현 대신 실제 상태이상 이름, 레벨, 발동 확률, 지속시간과 같은 hover 설명을 표시한다. 기존 property/material 태그와 forge material/trait/name signature가 후보 가중치를 만들지만, 종류·발동 확률·레벨·지속시간은 서버 난수를 크게 반영한다. 결과는 `enchantment`와 검증된 `attackEffects` metadata delta 및 해당 property 영속 태그로 저장된다. 근접과 아이템 투사체 기본 공격 모두 실제 피해가 들어간 뒤 같은 Item 공개 API를 호출하며, 독·화염 같은 후속 효과는 물리 타격 자체의 속성 상성을 바꾸지 않는다.

대장장이 메인 + 전사 서브가 Lv.200에 `전투 대장장이`가 되면 `무기 강화`를 얻는다. `/무기강화 <인벤토리 번호|장착칸>`은 `철근미궁 지핵 수정실`의 `강화 수정맥`에서만 채굴되는 `지핵 강화석` 1개를 소모한다. 강화는 실패·하락 없이 +5까지 진행되며 매 단계 공격력 또는 마법력과 무기 종류별 긍정 능력치를 누적하고, +5에는 주 공격 능력치 배율이 추가된다. 강화 단계와 modifier는 `reinforcement` metadata delta에 저장되므로 단조 결과·장인의 명명·마법 부여와 함께 유지된다.

대장장이는 루미나르 전직소의 전직관 세레나에게 다른 직업과 동일하게 선택할 수 있다. 메인 시험은 광맥 5개, 서브 시험은 광맥 10개를 파괴하며 메인 전직 시 철 곡괭이를 받는다. Lv.20부터 은맥 광부 보급소의 마도 대장장이 로안에게 받는 `불꽃 없는 제련법`도 별도 전직 경로로 유지된다. 어느 경로든 현재 사용할 수 있는 메인 또는 서브 직업 슬롯에 정식 1차 직업 `대장장이`가 배정되고 대장장이의 담금질·결 파쇄·마력 제련·금속 단조 스킬을 얻는다. 다른 메인 직업을 이미 선택했다면 서브 슬롯이 열리는 Lv.50까지 기다려야 하며, 두 슬롯이 모두 차 있으면 전직할 수 없다. 마력 제련은 별도 용광로·연료·대기열 없이 정신력을 사용해 인벤토리에서 가장 앞선 원광을 `2 + 스킬 레벨 + floor(제련 정밀도 × 10)`개씩 처리한다.

`/단조 <형태> <재료>`는 메인/서브/계보 중 대장장이 직업을 보유하거나 `금속 단조` 스킬을 별도로 보유하면 사용할 수 있다. 필요한 제련 소재를 확인하고 `forge_rhythm` 세션을 시작하며, 실패하면 선택 소재가 파손되고 성공하면 서버가 검증한 0~1 score를 장비 생성기에 직접 전달한다. 재료 power와 형태별 소모량으로 난이도 1~10, 박자 수, 정박·엇박·반 박자 연타의 조합을 확정한다. 난이도 4 이상은 단계마다 2.5%p의 품질 보정을 제공하되 실제 정확도가 성공 기준을 넘어야 적용된다. 감각에서 계산된 `제련 정밀도`는 최대 45%까지만 일반/Perfect 타격 판정창을 넓히며 결과 정확도를 직접 더하지 않으므로 실제 리듬 입력은 계속 필요하다. 클라이언트는 첫 입력 뒤 접근 cue와 망치 충격음을 합성하고 난이도·원 정확도·보정 품질을 분리해 표시한다. 성공 결과는 소재와 완성품을 `Inventory.replaceSelectedItems()`로 한 번에 교환하고 단조 이벤트·경험치·완성품 정보를 보낸다.

대장장이 생산 행동은 몬스터 처치 없이도 캐릭터가 성장하도록 현재 레벨의 `maxExp`에 비례한 경험치를 준다. 마력 제련은 처리 원광 하나당 `maxExp × 0.25%`, 1회 최대 4%를 지급한다. 단조 성공은 정확도와 재료 난이도로 `maxExp × (2% + 정확도 × 1.5% + max(0, 재료 power - 0.9) × 1%)`, 1회 최대 4%를 지급하며 실패에는 경험치가 없다.
