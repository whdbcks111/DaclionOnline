# 패치노트

사용자에게 공개할 일별 변경 기록은 `shared/patchNotes.ts` 한 곳이 소유한다. 서버의 `/패치노트` 명령과 클라이언트 `/patch-notes` 화면은 모두 `getPatchNotes/getPatchNote` snapshot API를 사용하므로 같은 내용을 중복 작성하지 않는다.

## 사용자 진입

- 햄버거 메뉴의 `패치노트`: 날짜 목록과 선택한 날짜의 분류별 변경 내용을 보여주는 반응형 계층 화면을 연다.
- `/패치노트`: 등록된 날짜를 최신순으로 모두 출력한다.
- `/패치노트 YYYY-MM-DD`: 특정 날짜만 출력한다. 날짜 자동완성과 `YYYYMMDD` 입력도 지원한다.
- 정보성 명령이므로 공개/비공개 모드에 따라 채널 공개 여부가 결정된다.

## 데이터 규칙

`PatchNoteCategory`는 콘텐츠·시스템·밸런스·편의성·오류 수정의 표시명을 소유하는 클래스형 enum이다. 각 날짜는 다음 필드를 가진다.

- `date`: `YYYY-MM-DD`
- `title`: 그날 변경의 짧은 제목
- `summary`: 한 문장의 전체 요약
- `sections`: 카테고리와 사용자 관점의 변경 항목

같은 날짜의 작업은 새 날짜 객체를 중복 생성하지 않고 기존 객체의 적절한 section에 합친다. `getPatchNotes()`가 날짜 역순 snapshot을 반환하고 내부 배열과 문자열 배열을 복사하므로 화면이나 명령이 마스터 데이터를 변경할 수 없다.

## 변경 완료 규칙

사용자에게 보이는 기능·콘텐츠·밸런스·오류 수정이 완료되면 같은 커밋에서 해당 날짜 패치노트를 갱신한다. 리팩터링, 테스트만 추가, 문서 오탈자처럼 플레이 결과가 달라지지 않는 내부 변경은 기록하지 않는다. 구현 세부 파일명이나 내부 key 대신 플레이어가 무엇을 새로 할 수 있고 무엇이 달라졌는지를 적는다.
