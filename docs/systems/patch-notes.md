# 패치노트

사용자에게 공개할 버전별 변경 기록은 `shared/patchNotes.ts` 한 곳이 소유한다. 서버의 `/패치노트` 명령과 클라이언트 `/patch-notes` 화면은 모두 `getPatchNotes/getPatchNote` snapshot API를 사용하므로 같은 내용을 중복 작성하지 않는다.

## 사용자 진입

- 햄버거 메뉴의 `패치노트`: 버전 목록과 선택한 버전의 배포일·분류별 변경 내용을 보여주는 반응형 계층 화면을 연다. 화면은 게임 전역 스크롤 잠금과 독립된 `100dvh` 세로 스크롤 컨테이너를 사용한다.
- `/패치노트`: 등록된 버전을 최신순으로 모두 출력한다.
- `/패치노트 1.0.0`: 특정 버전만 출력한다. 버전 자동완성과 선택적 `v` 접두사 입력도 지원한다.
- 정보성 명령이므로 공개/비공개 모드에 따라 채널 공개 여부가 결정된다.

## 데이터 규칙

`PatchNoteCategory`는 콘텐츠·시스템·밸런스·편의성·오류 수정의 표시명을 소유하는 클래스형 enum이다. 각 버전은 다음 필드를 가진다.

- `version`: SemVer 형식의 배포 버전. 저장값은 `1.0.0`, 표시는 `v1.0.0` 형식을 사용한다.
- `releasedAt`: `YYYY-MM-DD` 형식의 배포일
- `title`: 해당 버전 변경의 짧은 제목
- `summary`: 한 문장의 전체 요약
- `sections`: 카테고리와 사용자 관점의 변경 항목

같은 배포 버전의 작업은 새 객체를 중복 생성하지 않고 기존 버전의 적절한 section에 합친다. `getPatchNotes()`가 SemVer 역순 snapshot을 반환하고 내부 배열과 문자열 배열을 복사하므로 화면이나 명령이 마스터 데이터를 변경할 수 없다. 배포일은 버전을 대신하는 조회 key로 사용하지 않는다.

## 변경 완료 규칙

사용자에게 보이는 기능·콘텐츠·밸런스·오류 수정이 완료되면 같은 커밋에서 현재 배포 버전 패치노트를 갱신한다. 새 버전을 배포할 때만 새 버전 객체와 배포일을 추가한다. 리팩터링, 테스트만 추가, 문서 오탈자처럼 플레이 결과가 달라지지 않는 내부 변경은 기록하지 않는다. 구현 세부 파일명이나 내부 key 대신 플레이어가 무엇을 새로 할 수 있고 무엇이 달라졌는지를 적는다.
