# 미니게임·낚시 시스템

## 공용 미니게임 세션

미니게임은 서버가 `startMiniGame()`으로 userId에 일회성 session ID·검증 token·만료 시각·타입별 config를 발급하고 `miniGameStart`로 해당 사용자의 모든 소켓에 보낸다. 한 사용자는 동시에 하나의 미니게임만 실행한다.

클라이언트는 결과를 직접 확정하지 않는다. `MiniGameOverlay`가 입력 trace와 경과 시간을 `miniGameResult`로 돌려보내면 서버가 다음을 검사한다.

- 현재 Socket.io 세션의 userId와 미니게임 소유자
- 일회성 session ID와 24-byte token
- 서버 경과 시간, 타입별 제한 시간, 입력 개수·범위·시간 순서
- 타입별 validator가 원본 config로 재현한 실제 성공 조건

검증이 끝난 세션은 성공/실패와 관계없이 즉시 소비한다. 만료, 마지막 연결 종료, 명시적 기능 취소에는 `miniGameCancelled`를 보내며 타입 소유 기능의 정리 callback도 실행한다. 새 퍼즐·보스 패턴은 `shared/minigames.ts` DTO와 결정론 로직, `modules/minigame.ts`의 세션 validator, 클라이언트 renderer를 같은 변경에 추가한다.

`hazard_dodge`는 5초 동안 플레이어 토큰을 움직여 seed 기반 폭탄·가로/세로 레이저를 피하는 공용 보스 패턴 미니게임이다. `bombs`, `lasers`, `mixed` 모드를 지원하고, 보드 이동속도는 실행 시점 Player `speed`의 18배를 10~48 범위로 제한해 동기화한다. 관리자 패널의 테스트 카테고리는 일반/전설 낚시와 쉬운 폭탄·보통 레이저·고난도 복합 프리셋을 보상 없이 선택한 온라인 플레이어에게 실행한다.

## 낚시 흐름

1. `location:fishing` 태그 장소에서 손 슬롯에 `tool:fishing` 낚싯대를 장착한다. `/낚시` 실행 시 보조 슬롯에 `item:bait` 미끼가 없으면 `Inventory.findFirstItem()`으로 인벤토리 미끼를 찾아 묶음 전체를 보조 슬롯에 자동 장착한다.
2. `/낚시`는 낚싯대·장소·생존 상태를 검사하고 장착된 미끼 스택에서 하나만 소비한다. 남은 묶음은 보조 슬롯에 유지되고 `Equipment.count`로 저장된다.
3. 현재 행운·입질 속도·낚싯대 채집 능력을 snapshot하고 45~65초 기본 범위를 `fishingBiteSpeed`로 나눈 시간만큼 기다린다. 별도의 강제 최솟값은 없으며, 기본 능력치 1 + 초보자 낚싯대 0.1 + 지렁이 미끼 0.35 조합은 약 31~45초를 기다린다. 대기 notification은 남은 시간을 추측할 수 있는 길이 progress를 표시하지 않는다. 장소를 벗어나거나 낚싯대를 바꾸면 입질이 취소된다.
4. 입질 시 행운을 반영한 가중치로 일반/고급/희귀/서사/전설/신화 등급을 결정하고 progress 없는 경고 notification을 먼저 보낸다. 1초의 준비 시간이 지난 뒤 상태를 다시 검증하고 등급별 난이도의 `fishing_capture` 미니게임을 시작한다. `/낚시등급표`는 현재 장비·미끼·버프가 반영된 행운으로 같은 가중치 API를 계산해 실제 등급별 확률을 표시한다.
5. 성공하면 해당 등급 pool에서 물고기 하나와 등급별 범위 경험치를 지급한다. 인벤토리 중량이 부족하면 물고기는 현재 장소 바닥에 떨어진다.

기본 등급 가중치는 62/22/9/4.5/2/0.5다. 행운은 일반 비중을 줄이고 상위 단계일수록 더 큰 비율로 가중치를 높이며 0~100 범위로 제한해 계산한다.

등급별 상점 매입가는 일반 5, 고급 20, 희귀 90, 서사 400, 전설 1,800, 신화 8,000 Gold다. 포획 난이도와 희소성을 보상하도록 다음 단계마다 약 4~5배씩 가치가 상승하며, 가격 단일 원본은 `FishRarity.sellPrice`다.

## 포획 미니게임

게이지는 기본 50%에서 시작한다. 물고기가 채집 영역 안에 있으면 상승하고 밖에 있으면 하락하며 100%는 성공, 0%는 실패다. 클라이언트는 0~1 값을 좌측 기준 `scaleX`로 0~100% 전체 트랙에 표시하고, 현재 값이 낮으면 빨강에서 시작해 주황·초록을 거쳐 성공에 가까울수록 파랑이 되는 단색을 사용한다. 수면 보드·채집 영역·조이스틱도 테마 단색 면과 얇은 경계를 사용하며 장식성 gradient나 glow를 사용하지 않는다. 표시 숫자는 실제 100%가 되기 전 100으로 반올림하지 않는다. 물고기 경로는 seed·난이도·경과 시간으로 결정되어 서버와 클라이언트가 같은 위치를 계산한다.

- PC: WASD 또는 방향키
- 모바일/터치: 원형 아날로그 조이스틱
- 낚싯대 metadata: `fishingNetShape` (`circle | square | rectangle`)
- 능력치: `luck`, `fishingBiteSpeed`, `fishingNetSize`, `fishingNetSpeed`, `fishingGaugeStart`

클라이언트는 축 입력 변화를 서버 재생 해상도와 같은 20ms 구간별 마지막 값으로 합치고, 결과 전송 순간 경과 시간 안의 입력만 불변 snapshot으로 만든다. 서버는 일회성 session/token과 경과 시간을 먼저 확인한 뒤 입력 순서·미세한 시간 초과·축 범위를 정렬/clamp하고 과도한 trace는 최대 2,048개로 균등 축약한다. 이후 20ms 고정 간격으로 채집 영역 이동 제한, 물고기 포함 여부와 게이지를 다시 계산하므로 사소한 브라우저 입력 오차는 허용하되 클라이언트가 보낸 성공 boolean이나 임의 점수는 신뢰하지 않는다.

## 초기 콘텐츠

- 장소: 광장과 연결된 `루미나르 물빛 연못`
- 장비: 초보자·정교한 낚싯대와 보물상자 전용 범위 특화 `너울그물 낚싯대`·속도 특화 `급류바늘 낚싯대`, `/낚시` 시 묶음 자동 장착 `통통한 지렁이 미끼`
- 물고기: 일반·고급·희귀·서사·전설·신화 각 6종, 총 36종. `data/fishingCatalog.ts`가 아이템 정보와 낚시 보상 pool의 단일 원본이다.
- 상점: 루미나르 물빛 연못의 낚시상점에서 초보자·정교한 낚싯대와 미끼 판매, 등급별 물고기 매입

각 물고기는 `item:fish`, `property:water`, `fish_rarity:*` 태그를 가지며 상점은 개별 ID가 아니라 등급 태그로 매입한다. 따라서 카탈로그에 같은 등급의 물고기를 추가하면 낚시 pool과 상점 매입에 함께 반영된다. 낚시 아이템과 신규 능력치는 모두 대응하는 128×128 투명 캐주얼 아이콘을 가진다.
