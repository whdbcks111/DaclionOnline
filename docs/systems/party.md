# 파티 시스템

파티는 서버 프로세스 메모리에서만 유지되는 최대 5명의 온라인 협동 단위다. DB에 저장하지 않으며 마지막 연결 종료·명시적 나가기·강퇴·해산 때 즉시 정리된다.

## 수명주기와 공개 API

`modules/party.ts`의 `partyManager`가 내부 Party/초대 Map을 단독 소유한다. 외부 코드는 raw 멤버 배열을 수정하지 않고 다음 API만 사용한다.

- `invite/accept/decline`: 60초 초대 생성과 수락·거절. 파티가 있으면 파티장만 초대할 수 있고 수락 시점에도 정원과 파티장 상태를 재검사한다.
- `leave/disband/kick`: 나가기, 파티장 전용 해산, 파티장 전용 강퇴. 파티장이 나가면 가입 순서상 다음 멤버가 파티장이 되고 1명만 남으면 파티를 종료한다.
- `removeDisconnectedPlayer`: 마지막 소켓이 끊기거나 Player가 unload될 때 파티와 초대를 정리한다.
- `getParty/getHudData`: 변경 불가능한 구성 snapshot과 현재 생명력·정신력 HUD DTO를 반환한다.
- `areInSameParty`: 두 userId가 같은 파티인지만 반환해 PVP에서 아군 공격을 차단한다.
- `distributeMonsterExp`: 같은 장소의 생존 파티원에게 몬스터 경험치를 지급하고 개인별 적용 결과를 반환한다.

닉네임 또는 고유번호 대상 조회는 `playerRegistry.findOnlinePlayerByIdentity()`를 사용한다. 닉네임은 대소문자를 무시한 정확한 일치이고 고유번호는 양의 정수 또는 `#번호` 형식을 허용한다.

## 명령어

- `/파티초대 고유번호 또는 닉네임`: 온라인 비파티 플레이어에게 초대. 초대 메시지에 수락·거절 버튼을 제공한다.
- `/파티수락`, `/파티거절`: 가장 최근의 유효한 초대를 처리한다.
- `/파티정보`: 파티장, 레벨, 같은 장소 여부와 HP/MP를 표시한다.
- `/파티나가기`: 본인이 이탈한다. 파티장이면 파티장 권한을 이전한다.
- `/파티강퇴 고유번호 또는 닉네임`: 파티장만 같은 파티의 다른 멤버를 강퇴한다.
- `/파티해산`: 파티장만 전체 파티를 종료한다.

초대·수락·강퇴 같은 조작 결과는 정보 공개 모드와 관계없이 당사자에게만 보인다. `/파티정보`만 정보성 명령으로서 공개 모드의 영향을 받는다.

## 몬스터 경험치 공유

몬스터의 드롭과 골드는 기존처럼 마지막 공격원의 `attackOwner` 플레이어에게 지급한다. 경험치는 처치 시점에 같은 장소에 있고 생존한 파티원 모두가 몬스터의 기준 경험치를 받는다. 보상 참여자 중 최고 레벨과 본인 레벨의 차이에 다음 감쇠를 적용한다.

| 최고 레벨과 차이 | 경험치 배율 | 추가 상한 |
| --- | ---: | ---: |
| 0~9 | 100% | 없음 |
| 10~19 | 50% | 없음 |
| 20~29 | 20% | 없음 |
| 30 이상 | 10% | 본인 다음 레벨 요구 경험치의 10% |

다른 장소의 파티원과 사망·제압 상태의 파티원은 해당 처치 보상 계산에서 제외된다. 각 참여자는 받은 양과 레벨 차이 감쇠를 개인 보상 메시지로 확인한다.

## HUD

`PlayerStatsData.party`는 파티가 없으면 `null`, 있으면 파티원별 레벨·HP·MP·파티장·같은 장소 여부를 포함한다. `PartyHud`는 기본 표시되며 HUD 설정에서 위치·크기·투명도·표시 여부를 바꿀 수 있다. 다른 장소의 파티원은 흐리게 표시된다.
