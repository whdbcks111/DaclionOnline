# Chat Components Overview

`ChatMessage.tsx`가 `ChatNode` tree를 재귀 렌더링하고, 최상위 image가 있으면 원본 비율에 맞춰 메시지 본문도 함께 줄인다. 공개 메시지는 PC hover/focus와 모바일 터치에 답장 버튼을 제공하고 `replyTo`가 있으면 본문 위에 원문 전송자·요약 이동 카드를 표시한다. 필터 메시지는 서버의 `replyable: false`를 따라 버튼을 숨긴다. 서버가 Player 누적 플레이 24시간 미만으로 snapshot한 `newcomer` 메시지는 닉네임 바로 왼쪽에 `🌱`를, 전송 시점에 카르마 악명 단계 이상인 `karmaMarked` 메시지는 닉네임 바로 오른쪽에 `🥀`를 표시한다. `CommandAutocomplete.tsx`는 `utils/commandAutocomplete.ts`에서 판정한 슬래시 명령 및 첫 단어가 정확한 명령 별칭인 입력의 후보·키보드 선택 UI를 렌더링한다. `nodes/`에는 높이를 `34vh/320px` 이내로 제한하는 이미지, tooltip, 제목 선택형 구분선과 인터랙티브 월드 지도를 포함한 노드 타입별 renderer가 있다.

`WorldMapNode`는 방문 장소 각각을 독립된 색상 씨앗으로 만들고 지도 아래의 저해상도 Canvas에 연속된 바이옴 색상장을 렌더링한다. 장소 대표색은 씨앗 생성 시 밝기와 채도를 낮춰 넓은 영역에서도 원색이 과도하게 튀지 않게 한다. 각 픽셀은 모든 위치 씨앗까지의 거리 제곱에 완화값을 더한 역가중치를 적용해 색을 동시에 합산하며, 최근접 위치를 별도로 강조하지 않는다. 따라서 같은 색 위치들이 자연스럽게 힘을 합치고 다른 바이옴으로 갈수록 지도 전체에서 경계나 장소 주변의 인위적인 진하기 변화 없이 연속적으로 혼합된다. Canvas는 현재 viewBox와 함께 다시 그려져 이동·확대 후에도 빈 영역이 없다. 렌더 해상도를 화면의 30%로 제한하고 좌표와 완화 거리를 재사용해 이동·확대 중 연산량을 줄인다. 장소 이름은 실제 지도 viewport 너비로 SVG 단위를 역산해 PC와 모바일 모두 지도 확대율과 무관한 크기를 유지한다. 기본값은 9px이며 전용 `A−/A＋` 버튼으로 6px부터 12px까지 다섯 단계로 별도 확대·축소하고, 이름 표시 토글로 라벨만 숨기거나 다시 표시할 수 있다. 화면에서 라벨이 겹치면 현재 위치와 랜드마크를 우선으로 남기고 일부를 자동 생략하며 확대해 여유 공간이 생기면 다시 표시한다. 미방문 장소 자체의 색은 렌더링하지 않는다.

ChatNode union, 색상 token, 자동완성 메타데이터/선택 로직이 바뀌면 이 문서와 [`docs/systems/chat-command.md`](../../../../docs/systems/chat-command.md), 공유 타입을 갱신한다.
