# Pages Overview

`LocationEditor.tsx`의 장소 위험도는 `safe | neutral | hostile`를 모두 편집하며 중립/적대 노드를 서로 다른 평면 색으로 구분한다.

라우트 단위 화면과 해당 화면의 이벤트 wiring을 담당한다.

- `Login.tsx`, `Register.tsx`: 인증/이메일 인증 UI.
- `Home.tsx`: 채팅, 채널, 슬래시 명령·슬래시 없는 별칭의 명령/인자 자동완성, `@온라인닉네임` 귓속말 자동완성, HUD 데이터의 중심 조정자. `playerStats/locationInfo`는 `syncId/revision`을 비교해 오래되거나 중복된 snapshot을 버리고 전체 상태만 교체한다. 채팅 히스토리·채널 진입은 즉시 맨 아래를 표시하지만, 이후에는 하단 48px 안에 있을 때만 새 메시지를 즉시 따라가며 사용자가 과거 로그를 읽는 동안 자동 스크롤하지 않는다. 서버가 시작한 `MiniGameOverlay`도 마운트한다. 입력창 왼쪽 미디어 버튼의 다중 `image/*` 선택과 contenteditable의 클립보드 이미지 붙여넣기를 최대 10장 미리보기 대기열로 합치며, 개별/전체 삭제 후 전송할 때만 인증 HTTP 업로드와 다중 이미지 소켓 메시지를 실행한다. 전송 옆 공개/비공개 버튼은 서버의 정보 열람 모드와 동기화한다. 텍스트·이미지 메시지 전송 시 contenteditable 포커스를 유지해 모바일 키보드가 닫히거나 깜빡이지 않게 한다.
- `GameGuide.tsx`: 햄버거 메뉴에서 진입하는 반응형 계층 안내 화면. 텍스트 MUD 소개, 첫 10분, 실제 canonical 명령(`/상태창` 등)과 단축어, 캐릭터 성장, 탐험·전투, 상점·퀘스트·광질·제작·낚시, HUD와 채팅 편의를 장/문서 단위로 탐색한다.
- `LocationEditor.tsx`: 권한 10 사용자를 위한 위치 그래프·지도 랜드마크 `mapIcon`·대표색 `mapColor`·`namespace:path` 태그·`monster | resource` 통합 오브젝트·NPC ID 배치 편집기. 상단 뒤로가기는 관리자 페이지로 돌아간다.
- `AdminPage.tsx`: 권한 10 운영자의 온라인 우선 플레이어 목록, viewport 내부 스크롤 상세 검사, 자원별 전용 상태 bar, 카르마 값·단계 확인/설정, 단순 레벨 설정과 상승 시 기존 분배를 유지한 채 레벨당 전 스탯 +1·가용 포인트 +3을 지급하는 레벨 조정을 포함한 카테고리별 플레이어/월드 action, 보유 스킬 레벨 설정, 전체 채팅·알림 공지와 선택 플레이어 알림, 위치 편집기 진입. 별도 밸런스 탭은 검색 가능한 FormDialog로 스킬·직업·장비/버프 아이템 분석과 평타+전체 스킬 일반/보스 전투 로테이션 프로파일을 실행하고 긴 결과만 스크롤 Dialog로 표시한다.
- 각 `*.module.scss`: 해당 화면 범위 스타일.

라우트 흐름, 소켓 emit/on, 화면 책임이 바뀌면 이 문서와 [`docs/api/socket-events.md`](../../../docs/api/socket-events.md) 또는 관련 시스템 문서를 갱신한다.
